GETTING STARTED

Prerequisites

This repo holds primarily addons, plugins, scripts, and assets for use in Blender.
It is recommended to use this file suite with the most recent stable version of Blender.
Latest stable version Blender 3.1.2 - download Blender

Directory Setup

Before proceeding, make sure files are located in a secure, isolated location (not a Desktop or Downloads folder)

        Altering, renaming or adding to the contents of anything within the folder structure under
        the master folder ‘bpy’ may cause unexpected errors within some tools

Blender Setup

  • Open Blender and verify the version number currently being used via the splash screen.
  • Click Edit > Preferences from the upper toolbar
  • Navigate to File Paths in the preferences window
  • Under Asset Libraries click the + (add asset library)
  • Name this library whatever you’d like and click the folder icon to choose a directory
  • This directory must be set to your install location of the Library folder within the bpy folder, this will always end in ‘\bpy\Library\’
  • If you are using other personal asset libraries, this is fine and should not cause conflicts when adding this library
  • Now when an asset library window is open, you can use the Library dropdown (by default set to ‘Current File’), and set this to the name of the new library to access included assets.

ASSET LIBRARY OVERVIEW

The main advantage of the Asset Library is the fact assets are not added to the .blend until dragging and dropping into the scene. The assets live in blend files in the directory of the library. This means having access to an array of assets that do not bloat your working file size or crowd your dropdown menus until you add them to your working file.

Blender allows you to determine how assets are imported into your project:

  • Append - each time the asset is dragged and dropped from the browser, it will use the explicit data from the library.
  • For example, drag/dropping a material to two different objects using this method will create two new materials (materialname, materialname.001), and the same applies to any node groups, modifiers, meshes, etc contained in that library asset.
  • This can incidentally add many unwanted copies of data you may have intended to reuse (node_groups in materials and geometry nodes in particular)
  • Append (Reuse Data) - each asset dragged and dropped using this method shares the current file’s data for that asset.
  • For example, drag/dropping a material to two different objects using this method will use the same material on both of the objects. However, if this material is changed, the next time you drag/drop from the asset browser, the changed material will be used (not the original from the Library file)
  • Link - Similar to ‘Append’  but allows for upstream changes from the Library file
  • Not recommended to use in conjunction with the bpy/Library

Recommended Asset Library Workflow

  1. Add assets to scene using the Append (Reuse Data) method
  2. If changing the asset for any reason other than renaming (changing an image in a material, adding/adapting Geometry Nodes, etc )
  1. Duplicate (Shift+D) the object with data in question
  2. Select the original object you just duplicated
  3. Create a unique instance of the data by clicking the number next to the name (materials and node_groups), and name this something new
  4. Select the duplicate object
  5. Click the Shield Icon next to the material or node_group name (make fake user)
  6. Delete the duplicate object

Result:  Now the asset browser will continue to pull the unaltered data from the library, and duplicates of internal node_groups inside the data will not be unnecessarily duplicated.

Make Modules Accessible By Default

Note: If you currently have a custom startup file, follow instructions below for adding ToolManager to an existing project, then execute step 4 from this list.

  1. Find and open startup.blend in bpy.Templates
  2. Check the checkbox and click Always Allow when prompted upon opening the file
  3. In the 3D View Properties Panel UI (keyboard shortcut ‘n’ with mouse in 3D view window), there will be a new tab called ‘tools manager’ - do not make any changes inside the tools manager at this time
  4. Select File > Defaults > Save Startup File
  5. Select File > New > General, and do not save when prompted

Result: Now by default when opening a new blender file, the Tools Manager will be available to you with independent settings in each file.

Add Modules To Existing Projects

If you would like to make scripts accessible to a project you created previously, follow these instructions:

  1. In the .blend file you would like to add tools to, set your workspace to ‘Scripting’ via the workspace tabs at the top of the screen
  2. Click Text > Open (Alt+O) and navigate in your files to bpy/Library/Resources
  3. Select ‘ToolManager’ and click ‘Open’
  4. Run the script by pressing Alt+P or clicking the play button next to the name of the script
  5. Select Text > Register
  6. Change the workspace back to whatever you like
  7. Save the file (Ctrl+S)

Note: Registering the text will make this script accessible every time the file is opened

TOOL MANAGER OVERVIEW

Adding Modules Via ToolManager

Once ToolManager is accessible to you, and the bpy Asset library is linked in preferences, this process is very simple.

  • Select tools via checkboxes in the ToolManager panel
  • Click the Append Tools button in the ToolManager panel

Result: The selected tools and associated resources will be added to the project and made accessible each time this file is opened. Currently, every tool besides (GridPacker) will show as ToolManager does, as a tab in the Properties Panel of the 3D View window. The GridPacker panel is available at the bottom of the Output Properties Panel inside the Properties window (where you typically set render resolution, output path, etc)

Caution: Once added, tools currently cannot be uninstalled from the file without following a manual process of removing each of the scripts from the text editor and re-running the ToolManager script. This process is intended to be automated in future releases.

Caution: Close and re-open Blender rather than running File > Open from a file with any tools (other than ToolManager) installed to avoid known issues associated with a tool being ‘active’ but not including necessary resources. This is a high priority bug that will be remedied in future releases.
